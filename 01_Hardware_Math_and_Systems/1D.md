## 学习路径

### 推荐顺序

- 主线先看 `07 / 08`，把 CPU/GPU 协同和 CUDA/Triton 编程模型先立住；这两页都给每个 Q 配了最小验证代码，方便直接检查调度和映射直觉。
- 再看 `15 / 16 / 17 / 18 / 19`，把执行模型、stream、shared memory 和 fusion 的原理链路补齐，其中 `15 / 16 / 17 / 18 / 19` 已先改成 `Q` 结构的主线页。
- 最后看 `29 / 30 / 31`，把高级调度、动态 shape 和 MIG 的工程化理解补完。

### 面向后续部分

- 基础认知层：先把 `07 / 08` 看完，再进入 Part 3 的 Triton kernel 和系统调度主线，其中 `07 / 08` 的代码块主要负责把异构调度和编程模型的关键判断跑通。
- 执行补充层：`15-19` 主要帮助理解 Part 3 里的 block / mask / fusion 这条实现链路，其中 `15 / 16 / 17 / 18 / 19` 负责先把执行模型、shared memory、stream、block 组织和 fusion 讲深。
- 工程延展层：`29` 继续承接高级调度，`30 / 31` 则分别补动态 shape 和 MIG 这类更细的运行时判断。

## 环境说明

- 默认按 `CPU-first` 阅读，先把执行模型和调度关系想清楚；`07 / 08` 的 Q 都配了验证代码，适合边读边核对。
- `29` 更适合结合 GPU 环境理解调度细节，`15-19` 更适合阅读和推导；其中 `15 / 16 / 17 / 18 / 19` 已采用 `Q` 结构，先承接执行模型、shared memory、stream、block 组织和 fusion 主线。
- 如果要做工程验证，优先在具备 CUDA 工具链和可用 GPU 的环境里看。

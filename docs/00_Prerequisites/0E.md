# Group 0E: Debugging and Performance | 0E: 调试与性能

本组把 profiling、显存优化和调试技巧串成一条收口链，是 Part 0 的最后一层，也是后续所有实战页的共同底座。对 Part 2 来说，这一组的价值主要在于：当训练或推理代码跑不通时，你知道先看哪里、怎么拆、怎么定位、最后该改什么，而不是只会改参数碰运气。

## Group Overview | 组概览

这一组把“怎么测、怎么拆、怎么定位、怎么决策”说清楚，帮助你把性能意识和排错习惯变成可复用的工作方式。这里强调的是最小可操作方法：先用 profiling 看瓶颈，再用显存账本和异常定位把问题拆开，最后决定先改数据、算子还是训练骨架。

对 Part 2 来说，这一组最常补的语法桥是：`torch.profiler`、显存曲线、逐层定位、`cpu/gpu` 时间拆分、异常复现、梯度检查和最小化问题样本。它更像是你在 Part 2 遇到“跑得慢、爆显存、结果怪”时，先回头查观测结果，再往下拆问题的工具页。

## Learning Path | 学习路径

### Recommended Order | 推荐顺序

- 先看 [17](./17_PyTorch_Profiling_Basics.md) -> [18](./18_Memory_Profiling_and_Optimization.md) -> [19](./19_Debugging_and_Anomaly_Localization.md) -> [20](./20_Profiling_and_Memory_Ledger.md)，把 profiling、异常定位和显存账本先立住。若你在 Part 2 里会碰到训练变慢、显存不够或结果异常，这一组就是最实用的回看入口。

### Part 2 重点 | Part 2 Focus

- [17](./17_PyTorch_Profiling_Basics.md)：先补 `torch.profiler` 和瓶颈判断，解决“时间花在哪”的问题。
- [18](./18_Memory_Profiling_and_Optimization.md)：先补显存账本和优化路径，解决“显存去哪了”的问题。
- [19](./19_Debugging_and_Anomaly_Localization.md)：先补复现、定位和最小样本，解决“问题从哪来”的问题。
- [20](./20_Profiling_and_Memory_Ledger.md)：先补综合判断和优化决策，解决“下一步该改什么”的问题。

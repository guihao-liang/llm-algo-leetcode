## Learning Path | 学习路径

### Recommended Order | 推荐顺序

- 主线先看 [07](./07_CPU_GPU_Heterogeneous_Scheduling.md) 和 [08](./08_Programming_Models_CUDA_Triton.md)，把 CPU/GPU 协同和 CUDA/Triton 编程模型先立住；这两页都给每个 Q 配了最小验证代码，方便直接检查调度和映射直觉。
- 再看 [15](./15_CUDA_Execution_Model.md)、[16](./16_Warp_Block_SharedMemory_Basics.md)、[17](./17_CUDA_Stream_and_Asynchrony.md)、[18](./18_Triton_Block_Model.md) 和 [19](./19_Operator_Fusion_Introduction.md)，把执行模型、stream、shared memory 和 fusion 的原理链路补齐，其中这些页面都已改成 `Q` 结构的主线页。
- 最后看 [29](./29_CUDA_Stream_Advanced_Scheduling.md)、[30](./30_Dynamic_Shape_Handling.md) 和 [31](./31_GPU_Virtualization_and_MIG.md)，把高级调度、动态 shape 和 MIG 的工程化理解补完。

### Next Steps | 后续衔接

- 基础认知层：先把 [07](./07_CPU_GPU_Heterogeneous_Scheduling.md) 和 [08](./08_Programming_Models_CUDA_Triton.md) 看完，再进入 Part 3 的 Triton kernel 和系统调度主线，其中这两页的代码块主要负责把异构调度和编程模型的关键判断跑通。
- 执行补充层：[15](./15_CUDA_Execution_Model.md)、[16](./16_Warp_Block_SharedMemory_Basics.md)、[17](./17_CUDA_Stream_and_Asynchrony.md)、[18](./18_Triton_Block_Model.md) 和 [19](./19_Operator_Fusion_Introduction.md) 主要帮助理解 Part 3 里的 block / mask / fusion 这条实现链路。
- 工程延展层：[29](./29_CUDA_Stream_Advanced_Scheduling.md) 继续承接高级调度，[30](./30_Dynamic_Shape_Handling.md) 和 [31](./31_GPU_Virtualization_and_MIG.md) 则分别补动态 shape 和 MIG 这类更细的运行时判断。

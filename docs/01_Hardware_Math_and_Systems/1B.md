# 1B: 单卡硬件与访存优化

本组解决“单张 GPU 怎么算得快”的问题，核心是理解 GPU 架构、内存层次和 Attention 访存优化。

## 组概览

这一组先把 GPU 架构、内存层次和 Attention 的访存路径说清楚，是 Part 1 里“看见算子慢在哪里”的起点，也是后续相关优化页面的共同前提。阅读顺序和 Part 级前导路径见 [Part 1 导学](./intro.md)，从 [1A](./1A.md) 进入，再按需要继续到后续组页。

## 学习路径

### 推荐顺序

- 主线先看 `03 / 04`，把 GPU 架构、内存层次和 Attention 访存路径先立住，其中 `04` 负责讲清 KV Cache 和注意力显存优化，且只在最关键的原理点上配验证代码。
- 再看 `11 / 12 / 13 / 14`，把显存增长、混合精度、profiling 和 FlashAttention 的原理链路补齐，这几页是延展主线页。
- 最后按需要继续看 `23 / 24 / 25`，把 TensorCore、shared memory 和稀疏 Attention 的实现层理解补完；这三页都给前 3 个 Q 配了验证代码，第 4 个 Q 保留纯原理收口。

### 面向后续部分

- 基础认知层：先把 `03 / 04` 看完，再进入 Part 2 / Part 3 的算子、显存和性能分析页面，其中 `04` 负责把注意力显存优化的原理讲深，代码只覆盖最核心的可验证点。
- 执行补充层：`11 / 12 / 13 / 14` 主要承接后续训练、推理和 profiling 的共享直觉。
- 实现延展层：`23 / 24 / 25` 主要服务 Part 3 / Part 4 的 kernel、shared memory 和优化分析。

## 环境说明

- 默认按 `CPU-first` 阅读，先把数量级和账本算清楚。
- `03 / 04 / 23 / 24 / 25` 更适合在有 GPU 的环境里复现和验证，部分页面会标注 `GPU optional`；`04 / 23 / 24 / 25` 都是“部分 Q 有代码”的结构，重点代码只覆盖最关键的验证点。
- `11 / 12 / 13 / 14` 主要是延展主线页，默认以阅读和推导为主，重点看原理链路和小验证。
